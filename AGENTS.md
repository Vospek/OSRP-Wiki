# AGENTS.md — TI-Wiki

## What this is

Docusaurus 3.10 static wiki for **On-Site: Roleplay**, themed with a dark classified/bureaucratic aesthetic. Uses React 19, TypeScript 6, Tailwind CSS 4.

## Package manager

**pnpm** — despite the README mentioning `yarn`, the lockfile is `pnpm-lock.yaml`. Always use `pnpm`.

```bash
pnpm install
pnpm start          # dev server (hot reload)
pnpm build          # production build → build/
pnpm typecheck      # tsc (IDE-only, not used by docusaurus build)
pnpm serve          # serve the built site locally
pnpm clear          # wipe .docusaurus cache
```

Requires Node >= 20.

## Project structure

```
docusaurus.config.ts   # site config (v4 future flag enabled)
sidebars.ts            # autogenerated sidebar from docs/ folder
docs/                  # MDX doc pages (sidebar mirrors directory structure)
src/pages/             # standalone pages (index.tsx is homepage)
src/components/site/   # reusable themed components (barrel export in index.ts)
src/components/HomepageFeatures/  # homepage feature grid
src/css/custom.css     # massive dark-theme CSS (~1100 lines, all component styles)
src/css/tailwind.css   # @tailwind directives only
tailwind.config.js     # extended with custom colors (site, border, ink, faction, pl, status)
static/img/            # static assets
```

## Custom component library (`src/components/site/`)

All doc pages should use these components for visual consistency. Import from the barrel:

```tsx
import { PageHeader, SectionHeader, PageFooter, ArchiveBlock,
         QuoteBlock, WarningBox, Badge, StatusDot, InfoPanel, RelatedBar }
  from '@site/src/components/site';
```

Key types live in `src/components/site/types.ts`:
- `AccentColor` — `'gray' | 'red' | 'gold' | 'blue' | 'purple' | 'green' | 'orange' | 'black'`
- `StatusColor` — `'green' | 'yellow' | 'orange' | 'red' | 'redbio' | 'black'`
- `PLLevel` — `1 | 2 | 3 | 4 | 5 | 'x'`
- `BadgeVariant` — `'complete' | 'pending' | 'active' | 'vacant' | 'official' | 'redacted'`

CSS for these components is in `custom.css` sections 5–22 (not in Tailwind). Component class names follow BEM-ish patterns (e.g. `page-header`, `page-header--red`, `page-header__title`).

## Tailwind

Tailwind v4 with PostCSS. Custom tokens in `tailwind.config.js` define the project's design system — color names like `site-base`, `ink-primary`, `faction-lore`, `pl-4`, `status-green` are used in both JSX `className` and MDX. Dark mode via `[data-theme="dark"]` selector.

## Docusaurus config notes

- `future.v4: true` is enabled — some v3 defaults may differ
- `onBrokenLinks: 'throw'` — broken links fail the build
- Blog is configured in the preset but no `blog/` directory exists yet
- Sidebar is fully autogenerated from `docs/` directory tree (uses `_category_.json` files)

## Common pitfalls

- **Don't edit `.docusaurus/`** — it's generated cache, cleared by `pnpm clear`
- **Component styles are in `custom.css`**, not CSS modules — Tailwind classes and custom CSS classes coexist
- **`tsconfig.json` is IDE-only** — Docusaurus uses its own webpack config for builds; `ignoreDeprecations: "6.0"` is set for TS 6 compatibility
- **No blog directory yet** — creating `blog/` will activate the blog plugin automatically
- **Navbar/footer links still point to Docusaurus defaults** — update when customizing
